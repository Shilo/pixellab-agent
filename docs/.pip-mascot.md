# Pip Mascot Private Concept

This is a private design note for the PixelLab Pip mascot. Do not reference this file from `README.md`, `skills/pixellab-pip/SKILL.md`, marketplace metadata, or public-facing docs unless the mascot direction is ready to become part of the published project identity.

## Core Identity

Pip is a tiny Pixel Pup who helps agents fetch the right PixelLab route. He should feel helpful, practical, safe, and cheerful: a small workflow companion who knows whether to use MCP, REST v2, website/editor flows, Aseprite, Pixelorama, or legacy v1.

The mascot should read as a puppy first and a tool-routing assistant second. Avoid making Pip feel like a generic tech bot, wizard, or corporate logo mark.

## Recommended Style

Use a tiny rounded corgi-inspired scout puppy as the main direction.

- Compact rounded body with short legs.
- Oversized floppy ears for immediate cuteness.
- Warm honey-tan fur with a cream muzzle, chest, belly, and paws.
- Big curious dark eyes, but keep them simple enough for sprite readability.
- Bright cyan-blue collar as the main PixelLab/tooling accent.
- Small collar tag shaped like connected route nodes, a map pin, or a simple API graph.
- Friendly alert pose, ideally sitting with one paw raised.

The best tone is cute, practical, and trustworthy. Pip should look like he is ready to help, not like he is performing or selling something.

## Sprite Size Guidance

- `64x64`: primary mascot sprite size. This is the best balance for expression, collar detail, and readable silhouette.
- `32x32`: favicon, small UI status icon, or compact assistant marker. Simplify heavily.
- `96x96` or `128x128`: README, showcase, marketplace, and key art variants.

If animation is added later, start from the `64x64` sprite and build a small sheet:

- Idle sit.
- Paw raise.
- Fetch/run.
- Point/sniff route.
- Happy bark or success bounce.

## Production Sprite Notes

For the final sprite, simplify concept-art rendering into chunkier pixel clusters:

- Crisp outline.
- Limited warm palette plus cyan accent.
- No gradients, blur, shadows, or complex props.
- Keep the silhouette readable when scaled down.
- Make the route tag recognizable with only a few pixels.
- Avoid text and logos inside the sprite.
